# 网页游戏设计

为浏览器小游戏（vanilla HTML/CSS/JS，无引擎）设计有乐趣、上瘾、耐玩的游戏性，
并落地成可维护、可测、**可模拟验证**的代码。本 skill 是一套实战提炼的设计 + 数值 + 留存方法论。

## 设计哲学（先想清楚，再写）

两个方向收敛：

- **自上而下（体验）**：先问"玩家此刻该有什么感觉"。紧张？贪婪？惊喜？解脱？
  从理想体验倒推机制。任何不服务这个感觉的机制，砍掉。
- **自下而上（数值）**：从第一性原理推数值。掉率、成本、增长曲线必须能从
  "我想要的节奏"推导出来，不是拍脑袋——而且**先用电子表格/蒙特卡洛跑过再写码**（见 §3、§4）。

好游戏的机制让人觉得"本该如此"。说不出一个系统为玩家创造了什么决策或情绪，它就是噪音——删掉。

**三个上瘾引擎（每个机制都拿这三条照一遍）：**

- **随机奖励**：结果不确定才有期待。关键科学事实——**多巴胺峰值在"不确定的预期"阶段，
  而非兑现阶段**（可变比率 > 固定奖励，Hopson《Behavioral Game Design》）。固定奖励会被
  算计、会腻；可变比率（掉率/暴击/三选一/开箱）持续勾人。但随机必须有**保底地板**（见 §3），
  纯运气会劝退倒霉玩家。*自检：奖励可预测吗？可预测就不勾人——加方差，但给地板。*
- **即时反馈**：每个动作 <100ms 必须有回应（视觉/听觉/触觉）。反馈强度配奖励大小：
  小事轻响，大事（暴击/稀有揭示/被抓）才放大招（见 §6 juice）。*自检：点下去 0.1 秒内屏幕变了吗？*
- **心流体验**：难度紧贴能力、目标始终清晰、反馈即时不断，玩家才"忘记时间"。太难→焦虑，
  太易→无聊。消除一切打断心流的摩擦：加载、费解 UI、无谓等待、卡手动画、需要思考"现在点哪"。
  *自检：玩家任一刻都清楚"现在干嘛、为什么、结果如何"吗？*

同一台引擎：清晰目标 → 行动 → 即时反馈 → 随机奖励 → 更想再来一次。缺任一环，循环就泄气。

## 1. 核心循环 first

先定**核心循环**：玩家反复做的那件事 + 它的反馈 + 它喂养的成长。
```
[行动] → [即时反馈] → [资源/进度] → [变强/解锁] → 回到行动（但更强/更难）
```
循环本身要 30 秒内有快感，否则再多系统也救不了。先把循环跑通（哪怕丑），再往上加层。

## 2. 机制 = 决策张力

乐趣的核心是**有意义的选择**：每个选择有取舍，没有显然最优解。

- **Push-your-luck**（贪婪博弈）：继续 vs 收手。**关键：风险和收益要在一回合内同步爬升**——
  开局低风险低回报（多数人会继续）、中段中风险中回报（停在这里该被奖励）、再贪则"更小概率
  博更大回报"。没有失败概率就没有张力。被抓/爆掉损失累积收益（**局内真实损失**才让收益有分量）。
- **风险/收益对立**：让玩家在"稳"和"赌"之间选，两边都有人爱。
- **资源管理**：稀缺 + 多用途 = 决策。同一资源能升级 A 也能升级 B。
- **信息博弈**：隐藏信息 + 花代价揭示（鉴定/侦查）。赌它还是验它。
- **决策树 > 数值**：最好的解锁/道具**改变决策树**，不是单纯 +x%。一次性消耗道具
  （重抽/免伤/看穿）既是经济 sink 又是新决策，一石二鸟（见 §4、§5）。

反模式：无脑点击、唯一最优解、纯运气无决策。玩家不用思考就加一层取舍。

## 3. 数值平衡（可推导、可测、可模拟）

数值从目标节奏倒推，**种子化随机 + 纯函数让它可测**，并用**蒙特卡洛模拟验证再上线**。

**赌博机三参数（决定"体感"，要自觉调）：**
- **RTP/期望**：长期平均回报，定均值。
- **波动性（volatility）**：定"刺激曲线"——高均值 ≠ 常出奖，波动性才决定爽点起伏。指标 σ/μ。
- **命中率（hit frequency）**：任意回报的频率。保证"每局至少一次爽点"（SR+/暴击）的比例在目标区间。

**稀有度/掉率**：金字塔分布（普通多、传世少）。权重让目标稀有度的获取时间符合你要的"爽点间隔"。
各档池足够样本避免重复疲劳。**深入递增**：每深入一层后续高档爆率↑，激励冒险（贪婪有回报）。

**保底（pity，不是可选项，是留存命脉）**：纯运气让倒霉玩家流失。
- **软保底**：连续 N 次不出后，高档权重逐步陡升——**压缩霉运长尾**。
- **硬保底**：第 M 次必给。
- 关键认知：保底**塑造分布尾部、几乎不动均值**（可用模拟验证均值漂移 <5%、最长干旱被 M 封顶）。
- 公开概率/进度能降流失。pity 计数**跨局持久**（倒霉玩家跨局也受保护）。

**Near-miss（差一点）**：撤离/失败时揭示"你错过/留下的高档"——工程化的"差点赢"是最强"再来一局"钩子。

**实现范式 + 模拟验证**：
- 估价/掉率/判定写成纯函数，注入 RNG（`seeded(id)` 稳定 / `Math.random` 注入可测）。
- 全部可调数值集中在 `config.js`，调平衡不碰逻辑。
- 逻辑层 `node:test` TDD；分布跑 10 万次断言权重对。
- **蒙特卡洛脚本导入真实逻辑模块 + 真实数据池**模拟典型玩家，看均值/中位/命中率/波动性/霉运尾部，
  反调 config 到曲线达标（见 §4 经济体检同理）。**先模拟，再上线。**

## 4. 经济（source / sink，防通胀）

经济活在**流量**：每种资源要有入口（faucet/source）和出口（sink），平衡才不崩。

- **source 5 类**：Capped（一次性/封顶，如委托奖励）、Trickle（涓流）、Grind（刷）、Investment（投资）、Random（随机掉落）。
- **sink 4 类**：Fixed（一次性解锁）、Repeatable（可重复消费，如消耗道具）、Exponential（指数升级成本）、Competitive。
- **节奏**：早期 source > sink（让玩家爽、反哺成长）；中后期**必须有持续 sink**，否则钱贬值、目标真空。
- **指数成本**拉长曲线。终局靠一个**无限/指数 sink**（如声望 NG+ 晋升）吸收溢出——
  健康的"闲钱"其实是在攒下一次昂贵晋升的钱（有明确去向 ≠ 死钱）。
- **经济体检**：用真实 `state` 驱动**完整生涯模拟**（打局+委托+里程碑+收集+按策略花钱），
  看①各来源占比（别让某个 faucet 失控）②结余现金是否随时间加速堆积（sink 缺口）③解锁节奏。

## 5. 留存（治"玩一局就走"）

核心循环爽 ≠ 留存。Roguelite 留存几乎全靠 **meta 层**，四件套配齐：

- **每局不同（per-run 修正）**：每局随机一个 modifier（"今夜条件"），每个都是 **tradeoff 不是纯 buff**
  （如"高档涌现但警报更烈"）→ 制造新决策、破"每局一样"的腻。
- **跨局目标（委托/悬赏）**：销赃客户的订单横跨多局（"单局偷满 N"/"累计变现 X"/"连续不被抓"），
  达成发赏金 + 补新的 → **给"再来一局"的理由**，"就差一局把这单做完"。常驻 2-3 条、可折叠。
- **收集反哺玩法**：海量图鉴若无奖励=无底洞。加①**永久加成**（每收集稀有件永久抬售价，收集越多卖越贵）
  ②**里程碑赏金**（集齐 N 件一次性大额）→ 把无底洞拆成一串够得着的目标。
- **终局长线（声望/NG+）**：通关后花钱晋升，全局更烈更肥（无限指数 sink + 最高纪录/声望分）→
  给最强玩家造目标、长尾 aspiration。

原则：**解锁改决策树 > 加数值**；**局内真实损失**才让永久收益值钱；用 **near-miss** 拉下一局。

## 6. Game Feel / Juice（手感是第二个游戏）

同样的机制，有 juice 和没 juice 是两个游戏。反馈分层叠加：

- **即时**：点击有响应（按压/音效），每个动作 < 100ms 有视觉回馈。
- **揭示**：翻牌/开箱的延迟揭晓制造期待（**多巴胺在预期阶段**）——光柱预热掩盖加载 + 翻转揭示。
  可把"图片加载延迟"变成叙事（"撬锁中…侵入监控…"进度条），等加载完才揭晓。
- **稀有度表达**：越稀有越夸张（普通灰、传世金光粒子+流光+脉动）。
- **张力可视**：危险状态用整屏反馈（暗角 vignette、脉动渐红、抖动）——Balatro 式"氛围突变=视觉锚"，
  不看数字也知道风险模式变了。**注意：状态类加在容器上，离场要清，别漏到下一屏。**
- **奖励反馈**：数字 count-up 滚动、暴击金光爆、收集进度可见。
- **高光炸裂**：稀有揭示/大暴击/被抓加屏震，让"money shot"够爆（也利于短视频切片，见 §7）。
- **音效**：Web Audio 振荡器现场合成（零素材）。必须有静音开关 + 全程容错（`AudioContext` 不可用静默降级）。

juice 收敛原则：高 ROI 的做（被抓/暴击/稀有揭示），**全局乱闪和堆粒子是噪音，不做**。

## 7. 短视频 / 可分享（传播即增长）

从抖音/TikTok 视觉剪辑视角优化"可分享性"——自来水流量入口：

- **黄金 3 秒**：高光（稀有揭示/数字飙升）要够抓人，能当切片开头。
- **易剪易分享**：短促满足的视觉反馈、炫技壮举；鲜明画风 + 高对比才上镜。
- **9:16 上中三分之一**（15-40%）是视线落点，关键画面（主体、大数字）放这；远离顶/底（平台 UI 遮挡）。
- **高对比 + 大字号**；**金额缩写**（$1.05M / $313K，减认知负荷，别让一串数字劝退）。
- **可截图战绩卡** ⭐：一张漂亮的、带主画作 + 巨额 + logo + URL 的竖卡，截图直接发 → 最大传播杠杆。

## 8. 新手引导（教而不烦）

- **每屏首次触发**优于"跨屏动作跟随状态机"（后者脆弱）。首进某屏 → 高亮关键元素 + 一句话 →
  localStorage 标记不重复。
- 引导穿插在**做**里，不要一屏文字墙。第一次能偷时才教"偷"。
- 重置存档时清引导标记（重玩可重看）。

## 9. 内容数据驱动

- 内容存 JSON（`data/`），逻辑读数据，加内容 = 加数据不碰代码。
- 真实素材优先用免 key 开放 API（Met/芝加哥/克利夫兰博物馆艺术品、PokeAPI 等），代入感 > 占位图。
  并发常限流，**抓取走串行**。
- 图片运行时走 CDN URL（`<img>` 不受 CORS），元数据预构建成静态 JSON。
- **图片展示三层兜底**（防破图/空等）：进场预加载关键大图、慢图占位（叙事进度条）、所有图加载流光占位。
  彻底"不靠缓存也保证展示"则把字节镜像到自己的存储（R2 等），不依赖外站 CDN。

## 10. 落地架构（web 小游戏）

```
index.html          入口
js/
  config.js       ★ 所有可调内容/数值（平衡只改这里）
  <纯逻辑模块>.js   估价/掉率/判定/状态机/今夜条件/委托/收集/声望——纯函数，注入 rng，可 node:test
  ui.js           渲染 + 动效 + 交互（靠浏览器验收）
  audio.js        Web Audio 合成（容错）
  state.js        localStorage 存档（容错降级，私密模式不崩；旧档补新字段）
data/             内容 JSON
build/            蒙特卡洛模拟/经济体检脚本（导入真实逻辑验证数值）
test/             node:test 逻辑测试
```
- **可测的归逻辑层**（确定性纯函数），**手感/视觉归浏览器验收**。别给 UI 硬塞单测。
- 存档/音频/localStorage 一律容错：不可用静默降级，游戏不崩；存档加字段要给旧档兜底。
- 必须用 HTTP 打开（`fetch` 数据 + ES module 同源）。
- 改数值 → 跑模拟验证 → `node --test` 全绿 → 浏览器验收手感 → 上线。

## 自检（写完回头看）

- 核心循环 30 秒内有快感吗？三引擎在线吗（随机+保底 / <100ms 反馈 / 时刻清楚干嘛）？
- 每个机制创造了什么决策或情绪？说不出就删。
- **倒霉玩家有软+硬保底兜底吗？** 开局前 3 分钟够爽吗（首因效应）？
- push-your-luck 的风险和收益**同步爬升**吗？局内有真实损失吗？
- 留存四件套（每局不同 / 跨局目标 / 收集反哺 / 终局长线）配齐了吗？还是"一局就走"？
- 经济跑过**完整生涯模拟**吗？某个 faucet 失控吗？终局有持续 sink 吗？
- juice 是高 ROI 的还是为闪而闪？高光够"炸"能当切片吗？有可截图战绩卡吗？
- 状态类（危险红等）离场清了吗？有打断心流的摩擦吗（加载/费解 UI/卡手）？
- 加新内容只改数据吗？数值只改 config 吗？

逐条过，砍掉撑不起理由的东西。游戏设计是做减法。
